## 一、补齐目标

本次补齐基于毕设文档中的业务模型，将现有“用户、客户、板材、算法”基础能力扩展为可承载完整生产流程的数据结构：

- 余料管理：记录加工后可复用的板材余料。
- 订单管理：记录客户加工订单、客户快照、算法输入快照。
- 订单明细：记录每块待加工板件的尺寸、封边、材质和标签信息。
- 排版结果：保存算法排版结果 JSON、利用率、导出图片、刀轨文件和标签文件路径。

## 四、已补充的接口资源

| 模块 | 基础路径 | 说明 |

| 余料管理 | `/remnants` | 余料分页、详情、新增、编辑、删除 |

| 订单排版结果 | `/layout-results/order/{orderId}` | 查询某个订单的历史排版结果 |

## 五、实现细节说明

- `t_order_item` 补充了 `user_id`，用于兼容当前 MyBatis-Plus 租户拦截器，避免订单明细查询时缺少用户隔离字段。
- 新增实体的 `userId`、`createTime`、`updateTime` 继续通过 `AutoFillHandler` 自动填充。
- `TOrderService` 已实现订单与订单明细的事务保存；更新订单时，如果请求体传入 `items`，会替换该订单原有明细。
- `TLayoutResultService` 新增排版结果后，会自动回写 `t_order.layout_result_id`，方便订单快速读取当前采用的排版结果。
- MyBatis-Plus 日志实现已切换为 Slf4j，避免继续通过 stdout 输出 SQL。
