📖 Cutting System API 接口文档

项目名称：板材切割优化系统 (Cutting System)

版本：v1.0

最后更新：2026-03-16

一、基础信息

服务地址 http://localhost:8080
技术栈 Spring Boot 3.5.11 + Java 17
数据格式 JSON
字符编码 UTF-8

二、接口列表

1. 用户登录接口 /auth/login

功能说明：验证用户身份，登录系统

基本信息

请求路径 /auth/login
请求方法 GET 或 POST
Content-Type application/x-www-form-urlencoded 或 application/json

请求参数

参数名 类型 必填 说明
username String 是 用户名
password String 是 密码
请求示例（JSON 格式）
curl -X POST "http://localhost:8080/auth/login" \ -H "Content-Type: application/json" \ -d '{"username":"admin","password":"123456"}'

响应格式

字段名 类型 说明
code Number 状态码：200 成功，0 失败
msg String 响应消息
data Null 数据（登录成功时无数据）
成功响应示例
{ "code": 200, "msg": "success", "data": null }
失败响应示例
{ "code": 0, "msg": "用户名或密码错误", "data": null }

2. 算法求解接口（核心接口） /algorithm/answer

功能说明：接收装箱问题实例，返回最优放置方案

基本信息

请求路径 /algorithm/answer
请求方法 POST
Content-Type application/json

请求参数（InstanceDTO）

字段名 类型 必填 默认值 说明
L Number (double) 是 - 容器长度
W Number (double) 是 - 容器宽度
rotateEnable Boolean 否 false 是否允许物品旋转
gapDistance Number (double) 否 0 物品之间的间隔距离
squareList Array<Square> 是 - 待装入物品列表

Square 对象结构

字段名 类型 必填 说明
id String 是 物品唯一标识
l Number (double) 是 物品长度
w Number (double) 是 物品宽度
请求示例
curl -X POST "http://localhost:8080/algorithm/answer" \ -H "Content-Type: application/json" \ -d '{ "L": 100, "W": 50, "rotateEnable": false, "gapDistance": 0, "squareList": [ {"id": "item1", "l": 20, "w": 15}, {"id": "item2", "l": 30, "w": 25}, {"id": "item3", "l": 25, "w": 20} ] }'

响应格式

返回 SolutionResponseDTO 数组

字段名 类型 说明
containerLength Number (double) 容器长度
containerWidth Number (double) 容器宽度
rate Number (double) 空间利用率（百分比数值）
squareList Array<Square> 原始物品列表（同请求）
placeSquareList Array<PlaceSquare> 已放置物品的位置信息

PlaceSquare 对象结构

字段名 类型 说明
x Number (double) X 轴坐标（相对于容器左下角）
y Number (double) Y 轴坐标（相对于容器左下角）
l Number (double) 物品长度
w Number (double) 物品宽度
响应示例
[ { "containerLength": 100, "containerWidth": 50, "rate": 85.5, "squareList": [ {"id": "item1", "l": 20, "w": 15}, {"id": "item2", "l": 30, "w": 25}, {"id": "item3", "l": 25, "w": 20} ], "placeSquareList": [ {"x": 0, "y": 0, "l": 20, "w": 15}, {"x": 20, "y": 0, "l": 30, "w": 25}, {"x": 0, "y": 15, "l": 25, "w": 20} ] } ]
字段说明：
  • rate: 空间利用率，计算公式为 (所有物品面积之和 / 容器面积) × 100%
  • placeSquareList: 包含所有成功装入容器的物品位置信息，用于前端绘制
  • 如果物品无法全部装入，可能返回多个容器方案（数组形式）

三、统一响应结构

所有接口都遵循以下统一响应格式：

字段名 类型 说明
code Number 状态码
• 200: 请求成功
• 0: 业务错误
msg String 响应消息
data Object / Array / Null 响应数据（泛型）

四、错误处理

常见错误码

错误码 说明
200 请求成功
0 业务错误（如登录失败、参数错误等）
400 请求参数错误
404 接口不存在
500 服务器内部错误
错误响应示例
{ "code": 0, "msg": "错误描述信息", "data": null }

五、前端集成建议

1. Axios 请求封装示例

import axios from 'axios'; const api = axios.create({ baseURL: 'http://localhost:8080', timeout: 10000, }); // 登录接口 export function login(username, password) { return api.post('/auth/login', { username, password }); } // 算法求解接口 export function getAlgorithmSolution(instanceDTO) { return api.post('/algorithm/answer', instanceDTO); }

2. TypeScript 类型定义

// 请求数据类型 export interface Square { id: string; l: number; w: number; } export interface InstanceDTO { L: number; W: number; rotateEnable?: boolean; gapDistance?: number; squareList: Square[]; } // 响应数据类型 export interface PlaceSquare { x: number; y: number; l: number; w: number; } export interface SolutionResponseDTO { containerLength: number; containerWidth: number; rate: number; squareList: Square[]; placeSquareList: PlaceSquare[]; } export interface ApiResponse<T> { code: number; msg: string; data: T | null; }

六、注意事项

  • 跨域问题：当前后端未配置 CORS，如需前后端分离部署，请联系后端添加跨域支持
  • 坐标系：PlaceSquare 的 (x, y) 坐标是相对于容器左下角的绝对坐标
  • 单位：所有尺寸数据的单位由业务定义保持一致即可（如厘米、像素等）
  • 旋转功能：rotateEnable=true 时，算法会尝试旋转物品以获得更优解
  • 间隔距离：gapDistance 用于设置物品之间的最小间隔，默认为 0（紧贴放置）

输入数据
{ "L": 100, "W": 80, "rotateEnable": true, "gapDistance": 2, "squareList": [ {"id": "A", "l": 30, "w": 25}, {"id": "B", "l": 40, "w": 30}, {"id": "C", "l": 25, "w": 20}, {"id": "D", "l": 35, "w": 25} ] }
预期输出
[ { "containerLength": 100, "containerWidth": 80, "rate": 78.125, "squareList": [...], "placeSquareList": [ {"x": 2, "y": 2, "l": 30, "w": 25}, {"x": 34, "y": 2, "l": 40, "w": 30}, ... ] } ]
