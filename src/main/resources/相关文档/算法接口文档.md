## 二、核心接口详解

### 2.1 算法求解接口（核心业务接口）

#### 接口信息

- **接口路径**: `/algorithm/answer`
- **请求方式**: POST
- **Content-Type**: `application/json`

#### 功能说明

接收一个二维装箱问题实例，使用禁忌搜索算法计算最优放置方案，返回包含空间利用率和具体放置位置的解决方案。

#### 请求参数

**请求体格式**: `InstanceDTO` 对象

| 字段名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| `L` | double | ✅ 是 | - | 容器的长度（X 轴方向尺寸） |
| `W` | double | ✅ 是 | - | 容器的宽度（Y 轴方向尺寸） |
| `rotateEnable` | boolean | ❌ 否 | false | 是否允许物品旋转 90 度 |
| `gapDistance` | double | ❌ 否 | 0 | 物品之间的间隔距离 |
| `squareList` | Array | ✅ 是 | - | 待装入的物品列表 |

**squareList 数组元素结构**:

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `id` | String | ❌ 否 | 物品唯一标识（可不传，后端自动生成 UUID） |
| `l` | double | ✅ 是 | 物品的长度 |
| `w` | double | ✅ 是 | 物品的宽度 |

#### 请求示例

```json
 {
  "L": 100,
  "W": 50,
  "rotateEnable": false,
  "gapDistance": 0,
  "squareList": [
    {
      "id": "item1",
      "l": 20,
      "w": 15
    },
    {
      "id": "item2",
      "l": 30,
      "w": 25
    },
    {
      "id": "item3",
      "l": 25,
      "w": 20
    },
    {
      "l": 15,
      "w": 10
    }
  ]
}
```

#### cURL 测试命令

bash

```bash
curl -X POST "http://localhost:8080/algorithm/answer" -H "Content-Type: application/json" -d '{ "L": 100, "W": 50, "rotateEnable": false, "gapDistance": 0, "squareList": [ {"id": "item1", "l": 20, "w": 15}, {"id": "item2", "l": 30, "w": 25}, {"id": "item3", "l": 25, "w": 20} ] }'
```

#### 响应格式

**响应类型**: `List<SolutionResponseDTO>` （数组）

当物品需要多个容器才能全部装入时，会返回多个解决方案对象。

**SolutionResponseDTO 结构**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `containerLength` | double | 当前容器的长度 |
| `containerWidth` | double | 当前容器的宽度 |
| `rate` | double | 空间利用率（0-1 之间，越大表示利用率越高） |
| `squareList` | Array | 待装入的原始物品列表（包含所有物品） |
| `placeSquareList` | Array | 已放置的物品列表（只包含成功装入该容器的物品） |

**placeSquareList 数组元素结构**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `x` | double | 物品左下角的 X 坐标（相对于容器左下角原点） |
| `y` | double | 物品左下角的 Y 坐标（相对于容器左下角原点） |
| `l` | double | 物品的长度 |
| `w` | double | 物品的宽度 |
